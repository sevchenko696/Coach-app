# Project Guidelines

- Before writing code, think step-by-step and plan the approach
- Use production-grade practices (no shortcuts, no TODOs left behind)
- Follow clean architecture (separate concerns into layers: presentation, domain, data)
- Use TypeScript strict mode
- Add proper error handling (use try/catch, return meaningful error messages, never swallow errors)
- Validate all inputs at system boundaries
- Avoid code duplication (DRY principle)
- Use environment variables for configuration and secrets (never hardcode)
